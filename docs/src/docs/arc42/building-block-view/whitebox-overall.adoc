== Component description

|===
|Components |Description

|IRSApiConsumer
|A proxy for any consumer of the IRS API.

|IRS
|The IRS consumes relationship information across the CX Network and builds the graph view. Within this documentation, the focus is on the IRS.

|EDC Consumer
|The EDC Consumer Component is there to fulfil the GAIA-X and IDSA data sovereignty principles. The EDC Consumer consists of a control plane and a data plane.

|EDC Provider
|The EDC Provider Component connects to the EDC Consumer Component and forms the end point for the actual data exchange. It handles the automatic contract negotiation and the subsequent exchange of data assets for connected applications.

|Submodel Server
|The Submodel Server provides endpoints for requesting Submodel aspects.

|MIW
|Managed Identity Wallet as a self-sovereign identity provider for EDC.

|Discovery Finder
|The Discovery Finder is used to obtain the EDC Discovery Endpoint for a particular type of identity, e.g. BPNL.

|EDC Discovery
|The EDC Discovery Service is used to obtain EDC endpoints for a specific BPNL.

|Semantic Hub
|The Semantic Hub provides information about semantic models in a specific version. This includes JSON schemas used by IRS to validate the payload received from data providers.

|BPDM
|The Business Partner Data Management (BPDM) service is used by the IRS to access shared business partner data for BPNs of Data Providers along the supply chain.
|===
